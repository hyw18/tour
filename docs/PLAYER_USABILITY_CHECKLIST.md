# 플레이어 수동 사용성 체크리스트

최신 감사 기준: `5cf5b6a8edd0ac27f40efb262cec87cea381107c`  
대상: 게임을 처음 보는 참가자 최소 3명. 진행자는 조작법을 먼저 설명하지 않는다.

## 준비

- 실제 사용할 휴대전화와 브라우저로 호스트 1명, 플레이어 4명 게임을 연다.
- 참가자별 기기·브라우저·화면 방향·네트워크를 기록한다.
- 각 참가자에게 “한 턴을 진행해 보세요”만 말하고 타이머를 시작한다.
- 막혔을 때만 도움을 주고, 도움 요청 횟수와 내용을 기록한다.

## 참가자별 관찰표

| 측정 항목 | 참가자 1 | 참가자 2 | 참가자 3 |
|---|---:|---:|---:|
| 첫 행동까지 걸린 시간(초) | | | |
| 잘못 누른 버튼 수 | | | |
| 도움을 요청한 횟수 | | | |
| 한 턴 완료(예/아니요) | | | |

각 상황에서 `예/아니요`, 걸린 시간, 관찰 메모를 남긴다.

- 내 턴인지 3초 안에 바로 알 수 있는가?
- 지금 눌러야 하는 주 행동을 3초 안에 찾는가?
- “토지 구매 포기”와 “이번 방문 건설하지 않기”의 차이를 이해하는가?
- 건물 유형을 직접 선택하고 유형별 비용·수익 차이를 이해하는가?
- 도착 카드의 이번 방문 비용·수익과 과거 내역을 혼동하지 않는가?
- 자산·재무에서 자산, 세금, 대출, 최근 내역을 각각 찾는가?
- 자산을 선택했을 때 상세 확인과 “이 건물 관리하기”를 구분하는가?
- 거래 제안 전송과 실제 거래 성립을 구분하는가?
- 이벤트 확인 후 다음 행동을 찾는가?
- 오류 또는 연결 끊김 뒤 안내를 이해하고 복구할 수 있는가?
- 화면 회전 뒤 열린 팝업과 선택 상태가 유지되는가?
- 비활성 행동의 이유를 터치로 확인할 수 있는가?

## 통과 기준

- 3명 모두 첫 주 행동을 3초 안에 찾는다.
- 3명 중 2명 이상이 도움 없이 한 턴을 완료한다.
- 참가자당 잘못 누른 버튼이 1회 이하이다.
- 내부 오류 코드나 영문 예외가 노출되지 않는다.
- 실패한 참가자의 막힌 지점은 화면·문구·터치 문제로 분류해 재검증 항목에 남긴다.

## 실제 기기 필수 확인

- Android Chrome, Samsung Internet, iOS Safari에서 44×44px 터치 영역
- 가로·세로 회전, 모바일 뒤로가기, 키보드가 열린 상태의 팝업
- 네트워크 Wi-Fi 전환, 잠시 끊김, 백그라운드 복귀와 재접속
- 이벤트 확인 버튼의 화면 고정과 남은 시간 10초 이하 즉시 공개
- 장시간 게임 중 스크롤 위치, 발열, 애니메이션 프레임 저하
