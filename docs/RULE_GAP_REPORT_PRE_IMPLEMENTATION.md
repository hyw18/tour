## PARTIAL

- MONEY-002: 일반 경제 계산은 `Decimal`과 50,000원 `ROUND_HALF_UP`을 사용하지만
  이벤트 다중 배율은 중간 원 단위 반올림을 수행한다.
- RETURN-003: `explicit_override`는 지원하지만 엔진 전역 플래그로 남아 이벤트 소멸 후
  정상 범위 복귀가 보장되지 않는다.
- LOAN-005: 출발지 정산 현금은 자동상환하지만 대출 이후 발생하는 모든 세후 현금 유입에
  즉시 적용되는 일반 훅은 없다.
- USAGE-002: 승인자 목록과 응답은 구현됐으나 승인자별 독립 10초가 아니라 요청 전체가
  하나의 `created_at`을 공유한다.
- TRADE-003: 무건물 토지 거래에서는 소유자가 일치하지만 건물이 있는 권리 통합 거래가
  성립하지 않아 불변식 전체를 검증할 수 없다.
- EVENT-002: 허용 target은 로드되지만 `economic_growth`, `trade_balance`,
  `regional_economy` 일부는 실제 경제 계산 소비처가 없다. scope 인자가 `None`인 계산에서는
  personal/regional 효과 경계도 엄격하지 않다.
- BANKRUPTCY-003: 엔진 메서드는 있으나 실제 사람 플레이어가 인수 여부에 응답하는 일반
  플레이어 API/UI가 없다.
- EXIT-003: 주사위·턴 종료는 활동으로 기록하지만 승인·거절·관리 시도·이벤트 확인 전체를
  공통 활동 입력으로 기록하지 않는다.
- RANK-002: 동점 마지막 기준은 플레이어 ID 기반 결정론적 난수 1회이며 “서버 주사위 반복”
  절차가 명시적으로 구현되지 않았다.
- PRIV-001: 핵심 금융정보는 제거됐지만 공개 응답에 보드 설정, 건물 권리 체인,
  활성 요청 존재 등 공식 공개 목록 밖의 필드가 더 포함된다.

## 이번 작업의 허용 범위

1. 위 104개 규칙을 고유 ID와 함께 구조화 데이터로 등록한다.
2. 공식 문서와 구현·테스트 대응표를 생성한다.
3. 구조화 데이터의 스키마, 버전, 필수 수치, 기존 데이터 파일과의 일치성을 검증한다.
4. 위 `CONFLICT`·`UNRESOLVED`의 게임 동작은 사용자 승인 전 변경하지 않는다.
